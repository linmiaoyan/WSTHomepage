【把原 TeacherDataSystem / QuickVote「照搬」进本仓库的步骤】

1) 表结构
   - 打开原项目里的 .sql / 迁移脚本 / ORM 模型，确认表名与字段。
   - 将「CREATE TABLE …」合并进 server/schema.sql（注意与现有表重名时先改名或迁移）。
   - 修改 server/index.js 里对应的 INSERT / SELECT，使字段与原业务一致。

2) 已有 SQLite 文件
   - 将原 .db / .sqlite 复制到 server/legacy/（可自行建目录）。
   - 设置环境变量 WST_DB_PATH 指向该文件前，请先备份。
   - 若原库表名与当前 schema 不同：用 DB 浏览器或 sqlite3 做 INSERT INTO teacher_records … SELECT … FROM old_table。

3) 单进程 Flask（推荐）
   - 教师数据、问卷 API 已合并进根目录 server.py，与审批队列共用 keadmin_queue.db（另含 teacher_records / quickvote_* 表）。
   - 日常只需：python server.py（默认端口见文件末尾，常见 8000）。
   - 可选：本目录 Node（npm start）仅作备用；正式环境不必再启 Node。

4) 一键迁移（自动扫描 TeacherDataSystem / QuickVote 下 *.db）
   - 项目根目录双击 migrate.bat
   - 或: cd server && npm install && npm run migrate
   - 根目录默认 D:\OneDrive\09教育技术处，可 set MIGRATE_ROOT=你的路径
   - 强制重导: set MIGRATE_FORCE=1 && npm run migrate

5) 启动 API
   - 在 server 目录执行: npm install && npm start
   - 或使用项目根目录 start-api.bat
